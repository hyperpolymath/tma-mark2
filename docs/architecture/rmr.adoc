= RMR Utility - Responsibility Matrix Registry
:toc: left
:icons: font

== Overview

The RMR (Responsibility Matrix Registry) documents who is responsible for what within the eTMA Handler project. This utility provides a RACI-style matrix for project activities.

== RACI Legend

* **R** - Responsible (Does the work)
* **A** - Accountable (Final decision maker)
* **C** - Consulted (Provides input)
* **I** - Informed (Kept up to date)

== Development Responsibilities

[cols="3,1,1,1,1"]
|===
| Activity | Core Maintainers | Expert Contributors | Community | Users

| Feature Development | R/A | R | C | I
| Bug Fixes | R/A | R | R | I
| Code Review | R/A | R | C | -
| Testing | R | R | R | C
| Documentation | R/A | R | R | C
|===

== Escalation Paths

=== Technical Issues

1. Component Owner
2. Core Maintainers
3. BDFL (for deadlock)

=== Community Issues

1. Community Moderators
2. Core Maintainers
3. BDFL

=== Security Issues

1. Security Team (security@example.org)
2. Core Maintainers
3. External Security Advisor

== Updating This Registry

Changes to the RMR require:

1. Pull request with proposed changes
2. Review by Core Maintainers
3. 7-day comment period
4. Merge by BDFL or delegate
